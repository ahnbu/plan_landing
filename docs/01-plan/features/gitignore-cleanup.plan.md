# .gitignore 정리 및 Git 초기화 계획서

> 버전: 1.0.0 | 작성일: 2026-02-10 | 상태: 초안

## 1. 요약 (Executive Summary)
현재 프로젝트를 Git으로 관리하기 전에, 불필요한 파일과 디렉터리가 저장소에 포함되지 않도록 `.gitignore`를 최적화하고 정리합니다. 특히 Gemini CLI 작업 중 발생하는 임시 파일과 Stitch 프로젝트 관련 부산물들을 관리 규칙에 포함합니다.

## 2. 목표 (Goals and Objectives)
- 표준적인 Node.js/Vite 프로젝트 무시 항목 설정
- `.gemini/tmp` 등 AI 에이전트 작업용 임시 디렉터리 배제
- `stitch_project_*` 등 다운로드된 외부 에셋 관리 규칙 적용
- 깨끗한 상태에서의 Git 저장소 초기화 (`git init`)

## 3. 범위 (Scope)
### 포함 범위 (In Scope)
- `.gitignore` 파일 최적화 및 업데이트
- 현재 작업 디렉터리의 불필요한 파일 정리
- Git 초기화 전 최종 상태 점검

### 제외 범위 (Out of Scope)
- 원격 저장소(GitHub 등) 푸시 (추후 진행)
- CI/CD 파이프라인 구축

## 4. 성공 기준 (Success Criteria)
| 기준 | 지표 | 목표 |
|-----------|--------|--------|
| .gitignore 설정 | 누락 항목 여부 | node_modules, .gemini/tmp 등이 포함되지 않음 |
| 저장소 용량 | 초기 커밋 크기 | 소스 코드 위주의 가벼운 용량 유지 |
| 설정 완결성 | git status 결과 | 추적되지 않아야 할 파일이 목록에 나타나지 않음 |

## 6. 리스크 (Risks)
| 리스크 | 영향 | 완화 방안 |
|------|--------|------------|
| 필요한 파일의 오배제 | 개발 환경 복구 어려움 | 주요 설정 파일(`.env` 등)은 별도 관리 또는 템플릿 제공 |
| 대용량 파일 포함 | 저장소 성능 저하 | 이미지, 라이브러리 등은 LFS 사용 고려 또는 배제 |
