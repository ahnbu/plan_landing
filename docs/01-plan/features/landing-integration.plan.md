# 랜딩 페이지 섹션 통합 계획서

> 버전: 1.0.0 | 작성일: 2026-02-10 | 상태: 초안

## 1. 요약 (Executive Summary)
Stitch에서 다운로드한 5개의 개별 랜딩 페이지 섹션(`stitch_project_기획랜딩_멋쟁이style`)을 논리적인 순서에 따라 하나의 HTML 파일로 통합하여 전체적인 흐름을 한눈에 확인할 수 있도록 합니다.

## 2. 목표 (Goals and Objectives)
- 5개 섹션의 HTML 코드를 하나의 파일로 병합
- 사용자 경험 흐름에 맞는 섹션 순서 배치
- 공통 스타일(Tailwind 등) 및 스크립트의 충돌 방지 및 통합

## 3. 범위 (Scope)
### 포함 범위 (In Scope)
- `stitch_project_기획랜딩_멋쟁이style` 내 5개 폴더의 `code.html` 통합
- 통합 결과물인 `integrated_landing.html` 생성
- 섹션 순서 정의: Hero -> Problem -> Solution -> Curriculum -> CTA

### 제외 범위 (Out of Scope)
- 각 섹션 내부의 개별 컴포넌트 로직 수정
- 백엔드 연동

## 4. 성공 기준 (Success Criteria)
| 기준 | 지표 | 목표 |
|-----------|--------|--------|
| 통합 완결성 | 파일 생성 여부 | `integrated_landing.html` 파일이 정상적으로 브라우저에서 열림 |
| 시각적 일관성 | 섹션 배치 | 5개 섹션이 누락 없이 순서대로 표시됨 |
| 에셋 경로 | 이미지 표시 | 각 섹션의 이미지가 깨지지 않고 표시됨 (경로 조정 필요 시 수행) |

| 리스크 | 영향 | 완화 방안 |
|------|--------|------------|
| 스타일 충돌 | 레이아웃 깨짐 | 각 섹션을 고유 ID 또는 컨테이너로 감싸고 스타일 스코프 확인 |
| 이미지 경로 오류 | 이미지 미표시 | 상대 경로(`../`)를 통합 파일 위치에 맞게 수정 |
